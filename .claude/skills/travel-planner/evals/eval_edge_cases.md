# 评估用例：边界场景 + LLM 推理质量

## 场景 A：极端预算 + 画像约束冲突
```
我只有 500 块预算，和女朋友去南京玩 2 天纪念日。
从杭州出发坐高铁。希望吃得好住得好，但真的没钱。
```

### LLM 推理测试
- [ ] LLM 识别冲突：纪念日要氛围 vs 预算极低
- [ ] LLM 不因为冲突而放弃推荐，而是解释权衡：
  「纪念日出行，500 很紧。建议选这家民宿（¥200/晚）虽然房间小，
   但有露台可以看夜景，另外省下的钱吃顿好的。性价比路线而非硬撑高端」
- [ ] LLM 在推荐理由中坦诚预算紧张，不美化

## 场景 B：冷门目的地 + 少数据
```
我想去黑龙江鹤岗玩 3 天，帮我做攻略。
```

### LLM 推理测试
- [ ] 不拒绝冷门目的地
- [ ] cuisine-db 中无鹤岗数据 → LLM 诚实标注而非编造
- [ ] flyai 可能返回很少的酒店 → LLM 不捏造选项，在有限的选项中做最优判断
- [ ] 信息不足时标注「建议出发前核实」

## 场景 C：画像可被覆盖
```
（对话中已建立画像：不吃辣、怕高）
用户后来说：其实这次想试试重庆火锅，还有那个玻璃栈道也想去。
```

### LLM 推理测试
- [ ] LLM 不坚持约束（「但你之前说不吃辣…」← 不要这样）
- [ ] LLM 接受覆盖但标注风险：「重庆火锅辣度较高，建议从微辣开始。玻璃栈道注意安全，如果觉得不适可以走旁边的步道」
- [ ] 画像中其他未覆盖的约束仍然生效（如怕高被覆盖不影响其他约束）

## 场景 D：LLM 推荐自洽性审计
给定以下 LLM 输出，审计应该捕捉到矛盾：
```
画像：有老人膝盖不好，不吃辣
LLM 输出：推荐爬中山陵（392级台阶），午餐推荐川菜馆，推荐理由未提及老人和忌口
```
- [ ] Phase 5 审计标记：禁忌检查 FAIL（推荐了辣菜）
- [ ] Phase 5 审计标记：体力检查 FAIL（爬台阶 vs 膝盖不好）
- [ ] Phase 5 审计标记：推荐自洽性 FAIL（推荐理由未引用画像）

## 通过标准
- 所有 4 个场景的 LLM 推理质量达标
- 审计能正确捕捉推荐矛盾
- 画像覆盖行为正确（不固执，标注风险）
