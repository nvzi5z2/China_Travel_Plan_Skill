# 评估用例：家庭复杂旅程（画像驱动核心测试）

## 场景
2 个家庭（4 大人 + 2 小孩，小孩 5 岁和 7 岁），有老人膝盖不好，三亚 3 天。
从上海出发，需机票、酒店、租车。舒适优先预算，不吃辣，海鲜过敏。

## 输入
```
我们两家一起去三亚，4个大人2个小孩（5岁和7岁），还有我妈膝盖不太好。
7月15到17号，从上海出发。酒店要海棠湾的，预算每晚800-1500。
需要租车。对了我们都不吃辣，有个朋友海鲜过敏。
```

## 预期行为

### Phase 0：画像
- [ ] 出行组合：two_families, 4大2小, 有老人, 小孩年龄正确
- [ ] 饮食限制：no_spicy + shellfish 过敏
- [ ] 体力：有 mobility_issues (老人膝盖)
- [ ] 花钱风格：comfort_first

### Phase 1：需求收集
- [ ] 确认是否所有人同城出发
- [ ] 航班偏好：时段、直飞、舱位
- [ ] 酒店偏好：区域、类型、设施（亲子+无障碍）
- [ ] 租车需求确认

### Phase 2：调研
- [ ] flyai 搜索时不加 room-type 限制（宽搜索）
- [ ] flyai 搜索时不指定具体车型（宽搜索）
- [ ] flyai 返回多个酒店房型和多种车型供 LLM 筛选

### Phase 3：LLM 推理（核心测试）
- [ ] 酒店推荐需解释：为什么这个房型组合适合 2 个家庭 + 老人
- [ ] 租车推荐需解释：为什么这个车型适合 6 人 + 行李 + 老人上下车
- [ ] 餐厅推荐需标注：有没有辣、有没有海鲜、是否适合
- [ ] 每日行程有午休安排（小孩）、无爬山/长距离步行（老人膝盖）
- [ ] 每条推荐引用画像具体维度

### Phase 4：HTML
- [ ] 酒店卡片展示房型组合 + 推荐理由
- [ ] 租车卡片展示车型 + 推荐理由
- [ ] 飞猪预订链接可用

### Phase 5：审计
- [ ] 禁忌检查：无辣、无海鲜
- [ ] 体力曲线合理
- [ ] 健康安全标注
- [ ] 推荐自洽性：LLM 推荐理由引用画像，无矛盾

## 通过标准
- LLM 推理解释了「为什么推荐这个房型/车型/餐厅」
- 没有出现硬编码映射行为（如直接输出「GL8」而不解释）
- 画像约束全部在推荐中体现
- 没有任何推荐违反 dietary 或 mobility 限制
