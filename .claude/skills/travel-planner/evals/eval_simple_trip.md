# 评估用例：简单旅程

## 场景
单人 2 天北京，中档预算，美食为主，已订酒店（王府井附近），不需机票（同城）。

## 输入
```
帮我规划一个北京 2 日游，我一个人去，主要想吃好吃的。
酒店订在王府井的希尔顿，这周末出发。就在北京，不用订机票。
```

## 预期行为

### Phase 0：画像
- [ ] 提取：独自出行、美食为主、中档预算
- [ ] 画像不含特殊日子、无饮食限制、无体力问题
- [ ] 不预设任何推荐结论

### Phase 1：需求收集
- [ ] 一次一个问题，不是一口气全问
- [ ] 跳过交通模块（用户说不用订机票）
- [ ] 跳过住宿模块（用户已订酒店）
- [ ] 确认必做项目和兴趣标签

### Phase 2：调研
- [ ] 至少搜索 4 个维度（天气、酒店周边美食、北京美食热度、景点）
- [ ] 不调用 flyai 搜交通和住宿（用户不需要）
- [ ] 遵守反幻觉规则

### Phase 3：LLM 推理推荐
- [ ] 每条推荐有「为什么推荐给你」列
- [ ] 推荐理由引用画像维度（「一个人吃刚好」「中档预算范围内」）
- [ ] 没有硬编码推荐（不会因为「独自」而强制推荐单人套餐）

### Phase 4：HTML
- [ ] 纯 CSS + JS，零外部依赖
- [ ] 配色自动匹配（北京→皇城京韵主题）
- [ ] 无交通卡片（用户不需要）
- [ ] 无住宿卡片（已订酒店）

### Phase 5：审计
- [ ] 13 项审计全部执行
- [ ] 推荐自洽性检查通过（无矛盾）

## 通过标准
- Phase 0 画像正确提取
- 不搜索不需要的品类（不浪费 flyai 调用）
- 所有推荐有基于画像的解释
- 无硬编码推荐行为
